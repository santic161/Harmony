# agentic-decision — LLM Integration Reference

Use this file as context when generating code that integrates the `agentic-decision` library.
Paste the contents of this file into your LLM's system prompt or context window before asking it to write integration code.

---

## What the library does

`agentic-decision` runs multi-turn LLM agent loops that interact with real humans over messaging channels (Telegram, WhatsApp, Console, custom) until a validated, typed decision is reached. The result is returned as `Promise<Decision<T>>`.

- **Blocking:** `await orchestrator.startDecision(...)` — suspends until resolved
- **Non-blocking:** `.startDecision(...).then(handler)` or void — fires in background
- LLM speaks a strict JSON action protocol: `ask | propose | run_action | finalize | abort`
- Every LLM output validated by Zod. Invalid output → self-repair loop, not crash.
- User messages sanitized and wrapped in `<user_input>` delimiters before LLM sees them.
- Optional `skills` add task-specific instructions and examples only when enabled for a decision.
- Optional `actions` expose controlled runtime capabilities; the model can invoke only allowed action ids, never raw shell text.


## Core types

```typescript
// What startDecision resolves to
interface Decision<T> {
  status:     'finalized' | 'aborted' | 'timeout';
  value:      T | null;       // null unless status === 'finalized'
  confidence: number;         // 0–1
  reason?:    string;         // set on abort / timeout
  turns:      Turn[];         // full conversation history
  costUsd?:   number;
  sessionId:  string;
}

// LLM provider interface — any conforming object works
interface LLMProvider {
  name: string;
  generate(req: LLMRequest): Promise<LLMResponse>;
}

// Messaging channel interface
interface MessagingProvider {
  name:    string;
  start(): Promise<void>;
  stop():  Promise<void>;
  send(to: string, msg: OutboundMessage): Promise<void>;
  onReply(handler: (msg: InboundMessage) => void): () => void;
}

// State store interface
interface StateStore {
  get(id: string):                     Promise<SessionState | null>;
  save(id: string, s: SessionState):   Promise<void>;
  delete(id: string):                  Promise<void>;
}
```

## Main class: DecisionOrchestrator

```typescript
import { DecisionOrchestrator } from 'agentic-decision';

const orchestrator = new DecisionOrchestrator({
  llm:                 LLMProvider,             // REQUIRED
  messaging:           MessagingProvider[],     // REQUIRED — min 1
  store?:              StateStore,              // default: InMemoryStateStore
  logger?:             Logger,                  // default: pino info, PII-redacted
  costTracker?:        CostTracker,
  gate?:               ConfidenceGate,          // tune finalize threshold
  systemPrompt?:       string,                  // override base system prompt
  maxTurns?:           number,                  // default 20
  userReplyTimeoutMs?: number,                  // default 600000 (10 min)
  llmTimeoutMs?:       number,                  // default 30000
  llmTemperature?:     number,
  llmMaxTokens?:       number,
});

// Additional registries supported by the orchestrator:
// skills?: SkillDefinition[]        // global skill registry
// actions?: ActionDefinition[]      // global controlled action registry
//
// Each decision can opt into a subset with:
// skillIds?: string[]
// allowedActionIds?: string[]
//
// Start messaging providers
await orchestrator.start();

// Launch a decision
const decision = await orchestrator.startDecision({
  prompt:          string,    // REQUIRED — what the agent must decide
  userId:          string,    // REQUIRED — your internal user id
  channel:         string,    // REQUIRED — must match a registered provider's .name
  externalUserId:  string,    // REQUIRED — channel-native id (chat id, phone, etc.)
  schema:          ZodType<T>,// REQUIRED — Zod schema for decision.value
  metadata?:       Record<string, string | number | boolean>,
  kickoffMessage?: string,    // sent to user before first agent turn
});

// Shutdown
await orchestrator.stop();
```

## Controlled actions and skills

When `actions` are registered and a decision enables `allowedActionIds`, the LLM may emit:

```json
{ "action": "run_action", "actionId": "run_tests", "input": {}, "progressMessage": "Running tests now.", "reasoning": "Need fresh evidence." }
```

Important rules for generated integrations:

- Never let the model emit raw shell commands. Expose only named action ids.
- Validate `input` with Zod before execution.
- For shell actions, keep `command` fixed and derive args from validated input only.
- Feed the structured action result back into the loop before asking the human again.
- Use `skills` for contextual instructions/examples and `allowedActionIds` for hard runtime policy.

## Blocking vs non-blocking

### Blocking — await result

```typescript
// Code pauses here until agent finalizes, aborts, or times out
const decision = await orchestrator.startDecision({ ... });
console.log(decision.status, decision.value);
```

### Non-blocking — event-driven, fire and forget

```typescript
// Attach handlers before launching
orchestrator.on('decisionFinalized', ({ sessionId, decision }) => {
  saveResult(decision);
});
orchestrator.on('error', ({ sessionId, error }) => {
  console.error(error);
});

// Launch without awaiting — returns immediately
orchestrator.startDecision({ ... }).catch(console.error);
// Code continues here while agent runs in background
```

### Concurrent decisions

```typescript
// Multiple users simultaneously — each (channel + externalUserId) is isolated
const [a, b] = await Promise.all([
  orchestrator.startDecision({ externalUserId: 'user-1', ... }),
  orchestrator.startDecision({ externalUserId: 'user-2', ... }),
]);
```

### Custom channel

```typescript
import type { MessagingProvider, InboundHandler, OutboundMessage } from 'agentic-decision';

class MyProvider implements MessagingProvider {
  readonly name = 'my-channel';  // must match channel in startDecision
  private handlers = new Set<InboundHandler>();

  async start()  { /* subscribe / connect */ }
  async stop()   { /* unsubscribe / disconnect */ }

  async send(to: string, msg: OutboundMessage) {
    await myApi.sendMessage(to, msg.text);
  }

  onReply(handler: InboundHandler): () => void {
    this.handlers.add(handler);
    return () => this.handlers.delete(handler);
  }

  // Call from your platform's webhook / listener
  dispatch(userId: string, text: string) {
    for (const h of this.handlers) {
      h({ channel: this.name, externalUserId: userId, text, receivedAt: Date.now() });
    }
  }
}
```

## Zod schema — decision value

The `schema` parameter defines the TypeScript type and runtime validation of `decision.value`.
The agent is instructed to produce a `finalize.value` matching this schema.

```typescript
import { z } from 'zod';

// Simple approval
const schema = z.object({
  approved: z.boolean(),
  reason:   z.string().optional(),
});

// Ticket triage
const schema = z.object({
  priority: z.enum(['low', 'medium', 'high', 'critical']),
  owner:    z.string(),
  notes:    z.string().max(500).optional(),
});

// Free-form with discriminated union
const schema = z.discriminatedUnion('type', [
  z.object({ type: z.literal('approved'), signedBy: z.string() }),
  z.object({ type: z.literal('rejected'), reason:   z.string() }),
]);
```

## ConfidenceGate — tuning auto-finalize

```typescript
import { ConfidenceGate } from 'agentic-decision';

const gate = new ConfidenceGate({
  autoFinalizeThreshold: 0.98,   // skip confirmation at this confidence
  minConfirmThreshold:   0.6,    // below this, ask more questions
});

const orchestrator = new DecisionOrchestrator({ ..., gate });
```

## Security patterns

- Always cast SDK clients through `as unknown as OpenAILike` / `as unknown as AnthropicLike` etc. (structural interfaces).
- Never put user-controlled strings directly in `systemPrompt` — the guard handles isolation.
- Validate `decision.value` with your own business rules after it resolves — the agent is not a trusted source.
- Set `userReplyTimeoutMs` explicitly — default 10 min may be too long for your use case.

---

## Common errors

| Error | Cause | Fix |
|---|---|---|
| `No messaging provider registered for channel "X"` | `channel` in `startDecision` doesn't match any provider's `.name` | Check `provider.name` vs `channel` field |
| `Active session already running for X:Y` | `startDecision` called twice for same user before first resolves | Await or check before launching |
| `AllProvidersFailedError` | All LLM providers in FallbackChain failed | Check API keys and network |
| `TimeoutError: waitForReply` | User didn't reply within `userReplyTimeoutMs` | Increase timeout or handle `status: 'timeout'` |
| `OutputValidationError` after 3 repair attempts | Model consistently produces malformed JSON | Try a more capable model or simplify your Zod schema |

## Full minimal example

```typescript
import { z } from 'zod';
import OpenAI from 'openai';
import { DecisionOrchestrator, OpenAIProvider, ConsoleProvider, type OpenAILike } from 'agentic-decision';

const orchestrator = new DecisionOrchestrator({
  llm:      new OpenAIProvider({ client: new OpenAI() as unknown as OpenAILike }),
  messaging: [new ConsoleProvider({ externalUserId: 'user' })],
  userReplyTimeoutMs: 60_000,
});

const decision = await orchestrator.startDecision({
  prompt:         'Help the user pick a deployment strategy for their app.',
  userId:         'dev-1',
  channel:        'console',
  externalUserId: 'user',
  schema: z.object({
    strategy: z.enum(['blue-green', 'canary', 'rolling', 'recreate']),
    rationale: z.string(),
  }),
});

if (decision.status === 'finalized') {
  console.log('Chosen strategy:', decision.value?.strategy);
}

await orchestrator.stop();
```
